# 자료 작성 규칙 (메타데이터 & 청킹)

이 규칙을 지켜야 벡터검색이 정확해지고, 답변에 근거를 정확히 인용할 수 있다.
**새 자료를 넣기 전에 반드시 이 문서를 따른다.**

모든 md 파일은 상단에 YAML front matter(`---`로 감싼 영역)를 둔다.

---

## 1. 법령 (`corpus/laws/`)

```yaml
---
자료유형: 법령
법령명: 도시 및 주거환경정비법
법령ID: "001481"          # 국가법령정보센터 법령ID (개정 감지에 사용)
공포번호: "제19560호"
공포일자: 2023-08-08
시행일자: 2024-02-09       # ★ 개정 감지 기준
소관부처: 국토교통부
버전: 2024-02-09
출처: 국가법령정보센터 (https://www.law.go.kr)
최종확인일: 2026-07-22
---
```

### 본문 청킹 규칙
- 조문은 `## 제N조(제목)` 형식의 헤딩으로 시작한다. 검색이 조 단위로 자른다.
- 항은 `①②③…`, 호는 `1. 2. 3.…`, 목은 `가. 나. 다.…` 원문 기호를 그대로 쓴다.
- 별표·서식은 `### [별표 N] 제목` 헤딩으로 둔다.

예시:
```markdown
## 제39조(조합원의 자격 등)

① 정비사업의 조합원은 ... (원문 그대로)
② ...

## 제64조(재건축사업에서의 매도청구)
...
```

## 2. 판례 (`corpus/precedents/`)

```yaml
---
자료유형: 판례
사건명: 관리처분계획무효확인
사건번호: "2022두12345"    # ★ 인용·중복확인 기준
법원: 대법원
선고일자: 2023-05-11
관련법령:                 # 벡터검색 필터·연결에 사용
  - 도시 및 주거환경정비법 제64조
  - 도시 및 주거환경정비법 제39조
쟁점: 재건축 매도청구 행사 요건
출처: 국가법령정보센터 판례 / 대법원 종합법률정보
최종확인일: 2026-07-22
---
```

### 본문 권장 구성
```markdown
## 사실관계
...
## 쟁점
...
## 판단 (판시사항 / 판결요지)
...
## 실무 시사점   ← 회사 내부 해석 (선택)
...
```

## 3. 유권해석 (`corpus/interpretations/`)

```yaml
---
자료유형: 유권해석
제목: 조합설립인가 후 조합원 지위 승계 관련 질의
회신기관: 국토교통부 주택정비과
문서번호: "주택정비과-1234"
회신일자: 2024-03-15
관련법령:
  - 도시 및 주거환경정비법 제39조
질의요지: 조합설립인가 후 토지등소유자 변경 시 조합원 지위 승계 여부
출처: 국토교통부 / 법제처 국가법령정보센터
최종확인일: 2026-07-22
---
```

### 본문 권장 구성
```markdown
## 질의요지
...
## 회신내용
...
## 관련 법령
...
```

## 4. 공통 규칙

- **파일명**: 법령은 `법령명.md`, 판례는 `사건번호.md`(예: `2022두12345.md`),
  유권해석은 `회신일자_주제.md`(예: `2024-03-15_조합원지위승계.md`).
- **인코딩**: UTF-8. 한글 파일명 허용.
- **원문 보존**: 법령·판례·유권해석 본문은 **출처의 원문을 그대로** 옮긴다.
  요약·의역 금지(요약은 별도 `## 실무 시사점` 섹션에만).
- **날짜 형식**: `YYYY-MM-DD`.
- **추측 금지**: 확인되지 않은 내용은 넣지 않는다. 이 저장소의 자료가 곧 챗봇의
  "진실의 원천"이므로, 부정확한 자료 1건이 잘못된 답변을 만든다.

## 5. 템플릿 위치
- `corpus/laws/_TEMPLATE.md`
- `corpus/precedents/_TEMPLATE.md`
- `corpus/interpretations/_TEMPLATE.md`

새 자료는 템플릿을 복사해서 채운다.
