# 자료 추가 실무 가이드

새 법령·판례·유권해석을 챗봇에 반영하는 절차.

## 1. 자료 종류 판단
| 종류 | 넣을 폴더 |
|---|---|
| 법령·시행령·시행규칙·조례 | `corpus/laws/` |
| 판례 | `corpus/precedents/` |
| 유권해석·행정해석 | `corpus/interpretations/` |

## 2. 템플릿 복사
해당 폴더의 `_TEMPLATE.md`를 복사해 새 파일을 만든다.
파일명 규칙은 `docs/metadata-guide.md` 참조.

## 3. 메타데이터 채우기
front matter(상단 `---` 영역)를 규칙대로 채운다. 특히:
- 법령: `법령ID`, `시행일자` (개정 감지에 필수)
- 판례: `사건번호`, `관련법령`
- 유권해석: `문서번호`, `관련법령`

## 4. 본문 붙여넣기
**출처 원문을 그대로** 옮긴다. 요약·의역 금지.
- 법령: 조문을 `## 제N조(제목)` 헤딩으로 구분.
- 원문은 국가법령정보센터(https://www.law.go.kr) 등 공식 출처에서만.

## 5. (법령인 경우) 시행일 인덱스 갱신
`index/법령_시행일_인덱스.md` 표에 행을 추가/수정한다.

## 6. 커밋 & 반영
- 변경을 커밋해 저장소에 반영.
- 벡터 인덱스 재빌드(`scripts/build-index`)로 검색에 반영. *(도구 구현 후)*

## 개정 알림을 받았을 때
1. 알림에 표시된 법령의 최신 원문을 공식 출처에서 확인.
2. `corpus/laws/`의 해당 md 본문·메타데이터(`시행일자`,`버전`,`최종확인일`) 교체.
3. `index/법령_시행일_인덱스.md`의 시행일 갱신.
4. 커밋 & 인덱스 재빌드.
